- **App folder:** The production-grade app lives in `Task/` and uses Next.js (App Router) with TypeScript.

- **Components:** Reusable UI components in `components/` and `Task/components/`.
- **CI:** `.github/workflows/deploy.yml` builds & deploys `Task` with Vercel CLI.

**CI/CD Details**

- Workflow: `.github/workflows/deploy.yml` (runs on push to `main`). It now `cd`s into the `Task` folder, installs, builds, installs the Vercel CLI, and runs `vercel --prod` using the `VERCEL_TOKEN` secret.
- Required secret: **VERCEL_TOKEN** (more optional IDs: `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID`).

**Deployment Steps (recommended)**

1. Create a Vercel project and import the GitHub repository, or generate a Vercel personal token.
2. Add `VERCEL_TOKEN` to GitHub repo secrets: Settings → Secrets and variables → Actions.
3. Push to `main` to trigger the workflow, or use Vercel dashboard to deploy manually.

**UX & Feature Considerations**

- The `Task` app emphasizes scannability: large headings, card-based jobs, tag metadata, and skeleton loaders for fetch latency.
- Posting flow is intentionally simple for the assessment; consider adding authentication and moderation for production.

**Testing & Validation**

- Manual smoke tests:
  - Start dev server and verify landing page lists jobs.
  - Submit the post form and verify the job appears.
  - Check `GET /api/jobs` and `POST /api/jobs` behavior.
- Automated tests are not included; recommended next steps: add unit tests for critical components and API contract tests.

**Extensibility & Next Steps**

- **Persistence:** Integrate a database (e.g., Vercel Postgres, Supabase) and replace the module-level in-memory store.
- **Authentication:** Add OAuth or API key protected admin routes for posting jobs.
- **Search & Filters:** Add server-side indexing or use Algolia for scale.
- **File uploads:** Accept resumes/applications with direct S3 or Vercel Storage uploads.

**Security Notes**

- Current setup is a demo and not production hardened: no auth, no input sanitization beyond basic checks, and demo in-memory store. Do not use as-is for public production.
